# Пайплайн продакшна рекламных роликов по недвижимости

## Контекст
Первый ролик такого формата (`projects/32 sotki/`) собран вручную через одноразовые
ffmpeg-скрипты в скрэтчпаде. Будут ещё аналогичные заказы (другие участки/объекты) —
нужен переиспользуемый инструмент, а не разовый скрипт. У владельца есть подписка
CapCut, которую нужно задействовать для финальной полировки.

## Цель
Дать две дорожки сборки ролика из одного набора материалов (клипы, озвучка, ключевые
пункты объявления):
1. **Быстрая (ffmpeg)** — полностью автоматическая, без участия человека, годится
   когда нужен результат быстро или CapCut недоступен.
2. **CapCut-дорожка** — программно собранный черновик CapCut, который открывается
   в приложении уже с расставленными клипами/текстом/таймингом; финальную полировку
   (музыка из библиотеки, анимация субтитров, фильтры, цветокоррекция) и экспорт
   владелец делает сам в интерфейсе CapCut.

## Вне рамок (сознательно не делаем)
- Автоматический клик по интерфейсу CapCut (экспорт без участия человека) — библиотека
  `pyJianYingDraft` умеет это только для китайского интерфейса JianYing (ищет кнопки по
  тексту "导出" и т.п.), для англо/русского CapCut потребовалась бы отдельная разведка
  дерева UI Automation конкретной установленной версии — риск сломается при любом
  обновлении CapCut. Не берём в этот пайплайн.
- Брендинг (лого/фирменные цвета/аутро с контактами) — владелец подтвердил, что бренда
  пока нет. Каркас конфига оставляет место под это (branding: null), но не реализуем.
- Форматы кроме 9:16 — добавим отдельной задачей, когда реально понадобится.

## Расположение
Claude-скилл `~/.claude/skills/realestate-video-pipeline/`, по образцу существующих
(`24streets-data-pipeline` и т.п.) — подхватывается автоматически по ключевым словам
("рекламный ролик участка", "смонтируй видео объекта" и т.п.), не привязан к одной
папке проекта. Каждый конкретный ролик остаётся в своей `projects/{slug}/` (как
`projects/32 sotki/`) — скилл лишь содержит скрипты и документацию процесса.

## Общий конфиг проекта (`shotlist.yaml`)
Единый файл на входе для обеих дорожек, живёт в папке конкретного проекта
(`projects/{slug}/shotlist.yaml`):

```yaml
title: "32 сотки, Алматы"
aspect: "9:16"
clips:
  - file: "video/DJI_20260709090403_0001_D.MP4"
    in: 0.0
    out: 6.0
  - file: "video/DJI_20260709090617_0002_D.MP4"
    in: 2.0
    out: 8.0
  # ... порядок клипов = порядок в итоговом ролике
intro:
  file: "analog.MOV"
  in: 0.0
  out: 4.0
voiceover:
  file: "audio/voiceover_dmitry.mp3"
captions:
  style: "karaoke"       # karaoke | phrase | none
  source: "whisper"      # whisper (пословный тайминг) | srt (готовый .srt)
endcard:
  image: "territory.jpeg"
  duration: 5
  points:
    - "32 сотки в престижном районе Алматы"
    - "Выход к реке Есентай"
    - "Все центральные коммуникации"
    - "Высокий инвестиционный потенциал"
branding: null   # заготовка на будущее, см. "Вне рамок"
```

Ручной ввод `in`/`out` для клипов на этом этапе (автоматический подбор лучших
дублей — отдельная, более сложная задача компьютерного зрения, не в этой спеке).

## Дорожка 1: ffmpeg (быстрая, полностью автоматическая)

Скрипт `build_ffmpeg.py` в скилле, принимает путь к `shotlist.yaml`:

1. **Транскрипция и пословный тайминг** — `faster-whisper` (уже установлен) по
   `voiceover.file`, модель `small` (баланс скорость/точность на CPU), язык `ru`.
   Результат — список слов с `start`/`end` в секундах.
2. **Karaoke-субтитры** — генерация `.ass` с посегментной разбивкой (группировка слов
   в строки ~4-6 слов) и `\k`-тегами длительности каждого слова (стандартный karaoke-
   тайминг ASS/libass) — не просто прожиг готового `.srt`, а честная пословная
   подсветка, как в нативных TikTok/Reels субтитрах.
3. **Сборка**: конкатенация intro + всех clips (`trim`+`setpts` по `in`/`out` из
   конфига) + voiceover (с 4-й секунды, после интро) + endcard (image+drawtext по
   `points`), с fade-переходами между сегментами (как в текущей ручной версии
   `32 sotki`, но параметризовано конфигом вместо хардкода путей).
4. **Цветокоррекция по умолчанию**: лёгкий пресет `eq=contrast=1.05:saturation=1.15`
   — консистентный "чуть сочнее" вид на дрон-футаже без необходимости LUT-файлов.
5. Экспорт `final_cut.mp4` в папку проекта.

Никаких новых внешних сервисов — всё локально (ffmpeg через `imageio_ffmpeg`,
faster-whisper локально на CPU).

## Дорожка 2: CapCut-черновик

Скрипт `build_capcut_draft.py`, тоже принимает `shotlist.yaml`, использует
библиотеку `pyJianYingDraft` (ставится в venv скилла, не в системный Python):

1. `DraftFolder(<путь к CapCut Projects>).create_draft(name=...)` — создаёт новый
   черновик в `%LOCALAPPDATA%\CapCut\User Data\Projects\com.lveditor.draft\`,
   виден в списке проектов CapCut сразу при следующем открытии приложения.
2. Видеодорожка: `VideoSegment` на каждый клип из `clips` (+ intro) с `Timerange`
   по `in`/`out`, между клипами — `TransitionType` (простой crossfade по умолчанию).
3. Аудиодорожка: `AudioSegment` с `voiceover.file`.
4. Текстовые сегменты: `TextSegment` с таймингом под каждый пункт `endcard.points`
   на последних `endcard.duration` секундах — CapCut покажет их как редактируемые
   текстовые слои, которые владелец может доработать штатными инструментами CapCut
   (анимация, шрифт, стиль) прямо в интерфейсе.
5. Субтитры в CapCut-дорожке **не** генерируем через `pyJianYingDraft` — вместо этого
   в интерфейсе используется штатный "Авто-субтитры" CapCut (эта функция — то самое,
   ради чего подключаем платную подписку: у CapCut распознавание+анимация лучше и
   даёт больше стилей, чем наш ffmpeg karaoke-рендер).
6. Музыка, финальная цветокоррекция, фильтры — сознательно оставлены владельцу в
   интерфейсе (библиотека музыки CapCut требует интерактивного выбора, нет смысла
   гадать трек программно).
7. Экспорт — вручную, кнопкой в CapCut (см. "Вне рамок").

## Риски и ограничения
- `pyJianYingDraft` — сторонняя reverse-engineered библиотека (не официальный API
  CapCut/ByteDance). Формат черновика может измениться с обновлением CapCut и
  сломать генерацию — на такой случай дорожка 1 (ffmpeg) остаётся рабочим fallback
  независимо от версии CapCut.
- `faster-whisper` на CPU — модель `small` для ~1.5-минутного ролика займёт разумное
  время (десятки секунд/пара минут), не требует GPU.
- Конфиг требует ручной разметки `in`/`out` по клипам — на первом прогоне для
  нового объекта это самая трудоёмкая ручная часть.

## План внедрения / проверки
1. Написать скилл со скриптами двух дорожек.
2. Прогнать дорожку 1 (ffmpeg) на материалах `projects/32 sotki/` — сравнить с уже
   существующим `final_cut_32sotki.mp4`, подтвердить, что karaoke-субтитры реально
   лучше текущих phrase-субтитров.
3. Прогнать дорожку 2 (CapCut draft) на тех же материалах — открыть CapCut, убедиться,
   что черновик появился в списке и собран корректно (клипы на месте, текст на месте).
4. Задокументировать процесс в самом скилле (как заполнять `shotlist.yaml`, как
   запускать обе дорожки) — README скилла, не отдельный документ.
