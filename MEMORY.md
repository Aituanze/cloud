# Memory Index

Читать в начале каждой сессии для восстановления контекста.

## Записи

- [Профиль пользователя](memory/user_profile.md) — email, стиль общения, предпочтения
- [Правила работы](memory/feedback_workflow.md) — что одобрено / запрещено, как вести себя
- [Структура проекта](memory/project_structure.md) — папки, соглашения, backlog-процесс

## Соглашения по файлам

| Префикс | Назначение |
|---|---|
| `user_*.md` | Кто пользователь, его роль, стиль |
| `feedback_*.md` | Правила «как работать», одобренное / запрещённое |
| `project_*.md` | Контекст активных проектов, цели, бэклог |
| `reference_*.md` | Внешние ресурсы (дашборды, репо, трекеры) |

## Процесс

- **В начале сессии:** прочитать этот файл + нужные memory/*.md
- **В конце сессии:** если обнаружил новое знание о пользователе, проекте или правилах — предложить сохранить
- **Обновление:** не дублировать — сначала проверить существующие файлы
