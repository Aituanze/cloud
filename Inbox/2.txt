# No‑code / Low‑code прототип ИИ‑агента по поиску недвижимости (Казахстан)

**Цель:** за 4 недели создать рабочий прототип, который парсит объявления с krisha.kz (и других сайтов), показывает их в удобном интерфейсе с фильтрами и отправляет уведомления в Telegram. Всё без программирования, на готовых сервисах.

**Инструменты:**
- **Парсинг** → [Octoparse](https://www.octoparse.com/) (или [ParseHub](https://www.parsehub.com/))
- **Хранилище** → Google Sheets
- **Фронтенд** → [Glide](https://www.glideapps.com/)
- **Автоматизация уведомлений** → [Zapier](https://zapier.com/) (или [Make](https://www.make.com/))
- **Telegram‑бот** → [@BotFather](https://t.me/botfather) (создать без кода)

---

## Неделя 1: Настройка сбора данных (парсинг)

**Задача:** Настроить автоматический сбор объявлений с krisha.kz в Google Sheets.

### 1.1 Установка и регистрация
- Скачай и установи [Octoparse](https://www.octoparse.com/download) (бесплатная версия).
- Зарегистрируйся (аккаунт даёт 10 000 страниц в месяц – для прототипа достаточно).

### 1.2 Создание нового проекта
- Запусти Octoparse, нажми **«New Task»** → **«Advanced Mode»**.
- Вставь URL поиска на krisha.kz с нужными фильтрами, например:
https://krisha.kz/prodazha/kvartiry/almaty/?price[to]=25000000&rooms=2

text
(добавь свои параметры: район, цена, комнаты)
- Дождись загрузки страницы во встроенном браузере.

### 1.3 Выбор данных (извлечение полей)
- Наведи мышью на первую карточку объявления – появится окно с действиями.
- Выбери **«Select all»** (программа выделит все похожие карточки).
- Откроется панель **«Extract Data»**, добавь следующие поля (кликай по элементам на странице):
- **Заголовок**
- **Цена** (цифру)
- **Адрес** (район, улица)
- **Количество комнат**
- **Площадь** (общая, жилая)
- **Этаж / этажность**
- **Ссылка на объявление** (URL)
- **Первое фото** (img src)
- (опционально) **Описание** – для этого нужно перейти на детальную страницу, но для первого прототипа можно пропустить.

### 1.4 Настройка пагинации
- На странице найди кнопку «Следующая» (обычно внизу).
- В Octoparse нажми **«Loop»** → **«Click on element»** и выбери эту кнопку.
- Укажи максимальное количество страниц (например, 5), чтобы не перегружать парсер.

### 1.5 Запуск и экспорт
- Нажми **«Run»** (можно локально или в облаке).
- После завершения выбери **«Export Data»** → **«Export to Google Sheets»**.
- Авторизуй свой Google‑аккаунт и выбери таблицу (или создай новую). Данные будут загружены.

**Результат недели 1:**  
Google‑таблица с колонками: `Title`, `Price`, `Address`, `Rooms`, `Area`, `Floor`, `URL`, `Photo`.  
Новые объявления можно подгружать повторным запуском.

## Неделя 2: Создание интерфейса (приложение в Glide)

**Задача:** Сделать веб‑приложение, которое отображает объявления из Google‑таблицы, позволяет фильтровать и добавлять в избранное.

### 2.1 Регистрация в Glide
- Перейди на [Glide](https://www.glideapps.com/) и зарегистрируйся (бесплатный тариф).
- Нажми **«New App»** → выбери **«Start with a spreadsheet»**.
- Подключи свою Google‑таблицу с объявлениями.

### 2.2 Настройка макета
- Glide автоматически создаст карточки. Выбери вид **«List»** или **«Cards»**.
- Настрой отображение полей: фото (первая колонка с изображением), заголовок, цена, адрес.
- Добавь **детальный экран** – при клике на карточку показывай все поля и ссылку на оригинал.

### 2.3 Фильтры и поиск
- В левой панели Glide выбери **«Components»** → **«Search»** – добавь глобальный поиск.
- Добавь **«Filter»** для цены (слайдер) и для комнат (выпадающий список).
- Настрой, чтобы фильтры применялись к таблице.

### 2.4 Избранное (простой вариант)
- Добавь новый столбец в Google‑таблицу (например, `Favorite` типа `Checkbox`).
- В Glide на детальном экране добавь кнопку **«Save»**, которая при клике ставит галочку в этом столбце.
- Затем создай отдельный экран **«Favorites»**, который показывает только строки с `Favorite = true`.

### 2.5 Публикация приложения
- Нажми **«Publish»** → получи ссылку на веб‑приложение.  
- Она будет доступна на любом устройстве.

**Результат недели 2:**  
Работающее приложение, где риэлтор может просматривать объявления, применять фильтры и сохранять понравившиеся.

## Неделя 3: Добавление AI‑анализа (упрощённо)

**Задача:** Добавить текстовые метки (например, тип ремонта) и базовый анализ фото (если получится).

### 3.1 Текстовые теги (формулы в Google Sheets)
- В таблицу добавь столбец `Renovation_Type`.
- Воспользуйся функцией `REGEXMATCH`, чтобы определить наличие ключевых слов в заголовке или описании.  
Пример формулы (для ячейки `F2`, где в колонке `Description` описание):
=IF(REGEXMATCH(LOWER(F2); "дизайнерский|евроремонт|свежий"); "Дизайнерский"; IF(REGEXMATCH(LOWER(F2); "требуется|старый|без ремонта"); "Требует ремонта"; "Средний"))

text
- Скопируй формулу для всех строк (Google Sheets заполнит автоматически).
- Обнови Glide – добавь фильтр по этому столбцу (например, «Тип ремонта»).

### 3.2 (Опционально) Анализ фото через Zapier + Google Vision
> **Сложность:** средняя, потребуется аккаунт Google Cloud и настройка API. Если не готов – пропусти этот шаг, для MVP достаточно текстовых тегов.

**Шаги:**
1. Создай проект в [Google Cloud Console](https://console.cloud.google.com/) и включи **Vision API**.
2. Сгенерируй API‑ключ.
3. В [Zapier](https://zapier.com/) создай новый Zap:
 - **Trigger:** Google Sheets → New Row (срабатывает при добавлении нового объявления).
 - **Action:** Google Cloud Vision → Annotate Image (передай ссылку на фото из колонки `Photo`).
 - Получи ответ (теги, например, «kitchen», «living room»).
 - Ещё одно действие: Update Row in Google Sheets – запиши полученные теги в отдельную колонку.
4. Сохрани Zap и протестируй на нескольких объявлениях.

**Результат недели 3:**  
В таблице появляются AI‑метки (текстовые и, возможно, визуальные). Их можно использовать в фильтрах Glide.

## Неделя 4: Уведомления в Telegram и тестирование

**Задача:** Настроить автоматическую отправку сообщений в Telegram о новых объявлениях, подходящих под критерии.

### 4.1 Создание Telegram‑бота (без кода)
- Открой [@BotFather](https://t.me/botfather) в Telegram.
- Отправь `/newbot`, выбери имя и username.
- Получи **токен** (например, `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`).
- Напиши боту `/start`, чтобы активировать.

### 4.2 Настройка Zapier для отправки уведомлений
- В Zapier создай новый Zap:
- **Trigger:** Google Sheets → New Row (на каждое новое объявление).
- **Filter (опционально):** добавь условие, чтобы уведомление приходило только, если цена ≤ 25 000 000 и `Renovation_Type` = «Дизайнерский» (или любой другой критерий).
- **Action:** Telegram Bot → Send Message.
  - Выбери «Custom Bot» и вставь токен.
  - В поле «Chat ID» укажи свой Telegram ID (можно узнать у бота @userinfobot).
  - Сообщение составь, например:  
Новое объявление: {{Title}}
Цена: {{Price}} тг
Район: {{Address}}
Ссылка: {{URL}}

text
- Сохрани и включи Zap.

### 4.3 Тестирование
- Запусти парсинг в Octoparse, чтобы добавились новые строки.
- Проверь, приходит ли уведомление.
- Поделись ссылкой на Glide‑приложение с 2–3 знакомыми риэлторами.
- Попроси их пользоваться и записывать пожелания.

**Результат недели 4:**  
Полностью рабочий прототип: парсинг → таблица → интерфейс → уведомления. Готов к сбору обратной связи.

## Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Octoparse не видит элементы (изменилась вёрстка) | Перезапусти создание задачи в режиме «Advanced Mode», заново укажи элементы. Используй XPath вместо CSS‑селекторов. |
| Krisha.kz блокирует парсер (капча) | Увеличь задержки в настройках Octoparse (между страницами 5–10 сек). Попробуй использовать режим «Rotating User‑Agent». Если не помогает – попробуй ParseHub (иногда работает лучше). |
| Glide не обновляет данные сразу | В настройках источника включи «Auto‑update» (раз в час) или вручную обновляй таблицу. |
| Zapier не видит новые строки | Проверь, что триггер выбран «New Row» (а не «Updated Row»). Убедись, что таблица имеет заголовки. |
| Telegram не отправляет сообщения | Проверь токен и Chat ID. В Zapier в действии обязательно выбери «Test & Continue» для проверки. |

## Следующие шаги после MVP

Если риэлторам понравится, и ты получишь подтверждение спроса:

1. **Замени Octoparse** на собственный парсер на Python (с Playwright, ротацией прокси) – это повысит надёжность.
2. **Перенеси данные** из Google Sheets в PostgreSQL – ускорит работу с большими объёмами.
3. **Реализуй регистрацию** пользователей, разграничение доступа и личные избранные списки.
4. **Добавь полноценный AI‑анализ фото** (обучение модели на казахстанских квартирах) – это даст конкурентное преимущество.
5. **Интегрируйся с CRM** (REST API или готовый коннектор для AmoCRM, YClients).
