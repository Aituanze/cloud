# Krisha Parser — собственный парсер объявлений krisha.kz

## Описание

Собственный парсер на Python (замена no-code сервису Octoparse). Собирает карточки
объявлений со страниц поиска krisha.kz, **раскладывает данные по разрезам**
(район, категория, тип сделки, комнатность и др.) и сохраняет в SQLite
(`listings.db`) без дублей (по URL). Дополнительно выгружает всё в `listings.csv`
и печатает сводку по разрезам.

Это технический фундамент для направления [`realty-nocode-agent`](../realty-nocode-agent/README.md):
после проверки спроса парсинг переводится с Octoparse на этот собственный модуль —
ради надёжности и контроля.

> Стек: Python 3.10+, requests + BeautifulSoup, SQLite.

## Возможности

- Сбор карточек со страниц поиска krisha.kz по нескольким ссылкам (`SEARCH_URLS`).
- Разбор данных по разрезам:
  - **район** (`district`) — из адреса (районы Алматы + посёлки);
  - **категория** (`category`) — квартиры / дома / коммерческая / участки (из URL);
  - **тип сделки** (`deal_type`) — продажа / аренда (из URL);
  - **тип жилья** (`building_type`) — новостройка / вторичка (эвристика);
  - **комнатность** (`rooms`), **площадь** (`area`), **этаж/этажность**
    (`floor`/`total_floors`), **цена числом** (`price_value`).
- Сохранение в SQLite (`listings.db`) без дублей по URL.
- Выгрузка в `listings.csv` (UTF-8, открывается в Excel) и сводка по разрезам в лог.
- Настраиваемые задержки между запросами (ответственный парсинг).

## Структура данных (таблица `listings`)

| Поле | Описание |
|------|----------|
| `deal_type` | тип сделки: продажа / аренда |
| `category` | категория: квартиры / дома / коммерческая / участки |
| `rooms` | число комнат (`0` = студия) |
| `area` | площадь, м² |
| `floor`, `total_floors` | этаж и этажность |
| `district` | район Алматы / посёлок |
| `building_type` | новостройка / вторичка |
| `title`, `address`, `url` | исходные текст, адрес и ссылка |
| `price_text`, `price_value` | цена текстом и числом (тенге) |

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements.txt
```

## Запуск

1. Откройте `krisha_parser.py` и задайте список `SEARCH_URLS` — скопируйте ссылки
   поиска с krisha.kz с нужными фильтрами (район, цена, комнаты). Категория и тип
   сделки определяются автоматически из самой ссылки (по пути
   `/prodazha|arenda/` и `/kvartiry|doma|.../`).
2. Запустите:

```bash
python krisha_parser.py
```

Новые объявления попадут в `listings.db`, выгрузятся в `listings.csv`, а в логе
появится сводка по разрезам, например:

```
Категории → квартиры: 100
Типы сделок → продажа: 100
Тип жилья → вторичка: 88, новостройка: 12
Комнатность → 2: 100
Районы → Алатауский: 34, Наурызбайский: 25, Турксибский: 20, ...
```

## Важно

- **Селекторы** (`SEL_CARD`, `SEL_TITLE` и т.д.) в начале файла зависят от вёрстки
  сайта. Если ничего не парсится — откройте страницу в браузере (F12) и поправьте
  CSS-селекторы.
- Соблюдайте **задержки** (`REQUEST_DELAY_SEC`) и не делайте слишком много запросов.
- При капче/блокировках понадобится **Playwright** (браузерный рендеринг) и прокси.
- Используйте парсинг ответственно, соблюдая правила сайта и законодательство.
