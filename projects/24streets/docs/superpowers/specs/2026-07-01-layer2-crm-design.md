# 24streets Layer 2 — CRM + Buyer Window Design

**Дата:** 2026-07-01  
**Статус:** Утверждён

---

## Суть

Одна база, два окна. Агент видит рынок и ведёт сделки. Покупатель листает объекты в TikTok-стиле, регистрируется по телефону чтобы позвонить.

---

## Роли

| Роль | Доступ | Auth |
|---|---|---|
| Agency Admin | Оба окна + управление агентами | Email + invite |
| Agent | Только окно агента | Email + invite |
| Buyer | Только окно покупателя | Анон → телефон OTP |

---

## Окно агента

**Вкладки:**
- Карта — Pulse Map (без изменений)
- Объекты — мои объекты + добавить
- Сделки — CRM-воронка Kanban
- Профиль — настройки агента

**Поток: Собственник → Объект на платформе:**
1. Агент находит объект в Pulse Map / Архиве
2. "Взять в работу" → открывается форма Property Editor
3. Агент добавляет фото с камеры (до 10 штук), заполняет данные собственника
4. "Опубликовать" → объект появляется в ленте покупателей

**CRM-воронка (5 этапов):**
```
Новый лид → Контакт → Показ → Задаток → Сделка
```
Покупатель разблокировал телефон → автоматически создаётся карточка "Новый лид".

---

## Окно покупателя

**Вкладки:**
- Лента — TikTok-скролл опубликованных объектов
- Сохранённые — закладки
- Профиль — телефон

**Поток: Покупатель → Контакт:**
1. Открывает 24streets (анон) → видит ленту объектов
2. Нажимает "Позвонить" / "WhatsApp"
3. Экран: "Введите номер" → SMS OTP
4. Разблокирован: видит телефон агента + кнопку WhatsApp
5. Агент получает уведомление о новом лиде

## База данных (Supabase)

```
agencies        — агентства
profiles        — агенты/админы (auth.uid)
invites         — инвайты новым агентам
properties      — объекты (draft | active | archived)
buyer_profiles  — покупатели (auth.uid, phone)
leads           — CRM-воронка (new→contacted→showing→deposit→deal→lost)
lead_events     — история каждого лида
```

**RLS-принцип:**
- `properties` где `status='active'` → читают все (анон покупатели)
- `properties` полностью → только агенты своего агентства
- `owner_phone` в properties → агенты видят, покупатели нет (RLS на колонку через view)
- `leads` → только свои агенты

## Технический стек

- **Frontend:** ванильный JS (без сборщика, как сейчас)
- **Backend:** Supabase (PostgreSQL + Auth + Storage + Realtime)
- **Auth:** Supabase Auth (email для агентов, phone OTP для покупателей)
- **Фото:** Supabase Storage, сжатие client-side через Canvas API
- **CDN:** `@supabase/supabase-js` через jsDelivr
- **Хостинг:** GitHub Pages (статика + Supabase API)

## Разделение экранов по роли

```js
// В app.js при инициализации:
const { data: { session } } = await supabase.auth.getSession()
if (!session) → showBuyerWindow()
else if (profile.role === 'buyer') → showBuyerWindow()
else → showAgentWindow() // agent | admin
```
