# Realty Catalog — визуальный каталог объявлений

## Описание

Лёгкое веб-приложение (без сервера и сторонних сервисов), которое показывает
собранные парсером объявления **«по полочкам»**: автоматически раскладывает их по
районам, категориям, комнатности и типам, с фильтрами, поиском и пометкой
**«новое»**. Заменяет no-code интерфейс (Glide) из направления
[`realty-nocode-agent`](../realty-nocode-agent/README.md) собственной страницей.

Данные берутся из парсера [`krisha-parser`](../krisha-parser/README.md).

## Возможности

- **Подменю категорий с иконками** (в стиле `Inbox/icons.png`): Квартиры, Дома,
  Участки, Коммерческая недвижимость, Дачи.
- **Счётчик новинок за 24 часа над каждой иконкой** — сразу видно, сколько новых
  объектов появилось по каждой категории за сутки.
- **Погружение в категорию** — клик по иконке открывает категорию с **разбивкой
  по типам** (новостройка/вторичка), можно переключить на районы/комнатность.
- **Только первые вхождения, без дублей** — повторяющиеся объявления (по ссылке)
  в каталог не попадают; новизна считается по моменту первого появления.
- **Фильтры** (район, цена), **поиск** (адрес, район, ЖК), **сортировка**
  (новинки/цена/площадь).
- Светлый, простой интерфейс; карточка с ценой, параметрами и ссылкой на krisha.kz.

## Как считаются «новинки за 24 часа»

- Если в данных есть поле `first_seen` (его проставляет парсer при первом
  обнаружении объявления) — используется оно.
- Если поля нет — каталог сам запоминает в браузере момент, когда впервые увидел
  ссылку, и считает объявление новым 24 часа с этого момента.

## Структура проекта

```
realty-catalog/
├── index.html        # страница
├── css/styles.css    # светлая тема, адаптив
├── js/icons.js       # SVG-иконки категорий + список категорий
├── js/app.js         # логика: категории, новинки 24ч, фильтры, группировка
├── data/listings.js  # данные (генерируются из базы парсера)
├── build_data.py     # генератор data/listings.js из ../krisha-parser/listings.db
└── README.md
```

## Запуск

Просто откройте `index.html` в браузере (двойной клик) — приложение работает
локально, без сервера.

## Обновление данных

1. Соберите свежие объявления парсером:

```bash
cd ../krisha-parser
python krisha_parser.py
```

2. Сгенерируйте данные для каталога:

```bash
cd ../realty-catalog
python build_data.py
```

3. Обновите страницу в браузере.

> Файл `data/listings.js` можно обновлять только через `build_data.py` —
> вручную его править не нужно.
