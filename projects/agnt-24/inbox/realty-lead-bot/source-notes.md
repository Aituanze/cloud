# Исходные заметки: Telegram-бот для сбора лидов (из inbox)

> Перенесено из `inbox/1.txt`. Это первоисточник — готовый пример проекта бота
> для сбора заявок по недвижимости с сохранением в Google Sheets и уведомлениями
> в Telegram-группу.

## Идея

Telegram-бот принимает запросы от клиентов на недвижимость, сохраняет их в
Google Таблицу и уведомляет группу риэлторов. Объекты подбираются **вручную**
(без парсинга) — это полностью легально.

## Структура (из заметки)

```
/мой_бот
├── bot.py               # основной код бота
├── config.py            # настройки (токен, имя таблицы)
├── requirements.txt     # зависимости
├── credentials.json     # ваш файл от Google API (скачайте сами)
└── README.md            # инструкция по запуску
```

## config.py (пример из заметки)

```python
TELEGRAM_TOKEN = "ВАШ_ТОКЕН_БОТА"          # получите у @BotFather
GOOGLE_SHEET_NAME = "Лидогенератор"        # имя вашей Google таблицы
TELEGRAM_GROUP_ID = "@ваша_группа"         # ID группы для уведомлений
```

## Ключевая логика бота (из заметки)

- Подключение к Google Sheets через сервисный аккаунт (`credentials.json`).
- `/start` — приветствие, просьба описать запрос (район, комнаты, бюджет, телефон).
- На каждое сообщение клиента — строка в таблицу:
  `Дата, ID, Имя, Фамилия, Username, Запрос, Статус`.
- Уведомление в группу риэлторов: «🔥 Новый лид!» + данные + назначение риэлтора.

## Инструкция по запуску (из заметки)

1. Установить Python 3.8+.
2. `pip install -r requirements.txt`.
3. Настроить Google Sheets: Google Cloud Console → включить Sheets API →
   сервисный аккаунт → скачать `credentials.json` → создать таблицу
   «Лидогенератор» с колонками (Дата, ID, Имя, Фамилия, Username, Запрос, Статус)
   → расшарить таблицу на email сервисного аккаунта.
4. Настроить бота: @BotFather → `/newbot` → токен в `config.py`. ID группы — через
   @getidsbot.
5. Запуск: `python bot.py`.
6. Запуск на Android — через Termux (`pkg install python python-pip`).

## Важные замечания (из заметки)

- Парсинг krisha.kz **не используется** — объекты подбираются вручную (легально).
- `credentials.json` секретный — не публиковать.
- Идеи развития: команда `/assign` для назначения риэлтора, голосовые сообщения,
  подключение API krisha.kz при появлении официального доступа.
