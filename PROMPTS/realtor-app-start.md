# Старт: Приложение для риэлторов

## Контекст
Я риэлтор из Алматы (6+ лет опыта). Хочу создать веб-приложение для риэлторов —
красивое, современное, удобное и информативное.

## Что нужно сделать перед стартом

1. Прочитай BACKLOG.md в корне проекта
2. Запусти скилл `superpowers:brainstorming` — проведи разведку требований
3. После brainstorming — запусти `superpowers:writing-plans` и составь план

## Стек под капот (настрой до кода)

- [ ] Добавить в `.cursorrules` правило `frontend-design` с cursor.directory
- [ ] Подключить MCP `@modelcontextprotocol/server-google-maps` (карты и геокодинг)
- [ ] Опционально: MCP `@modelcontextprotocol/server-puppeteer` (парсинг Krisha.kz)

## Ключевые требования к приложению

- Дизайн: современный, привлекательный, мобильный
- Аудитория: риэлторы (не клиенты) — инструмент для работы
- Карты: обязательно (локации объектов)
- Язык: русский интерфейс

## Вопросы, которые brainstorming должен закрыть

1. Какие задачи риэлтор решает ежедневно? (показы, CRM, объекты, клиенты?)
2. Что самое болезненное в текущих инструментах?
3. Мобайл-первый или десктоп?
4. Нужна ли база данных / авторизация?
5. Интеграция с внешними площадками (Krisha.kz, OLX)?
